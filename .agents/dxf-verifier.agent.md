# DXF Verifier Agent

**Role:** Verify generated DXF line drawings against source product photos using Gemini 2.5 Flash vision API.

**When to use:**
- After any DXF is generated by a `save_*()` function
- When you need a semantic quality check: does the DXF actually look like the product?
- When you want specific, human-readable issues (e.g., "legs are too short", "missing TOP VIEW")

**How it works:**
1. Takes the source product photo + rendered DXF as two images
2. Sends both to Gemini 2.5 Flash with a structured verification prompt
3. Gemini returns: shape_match (0-1), component_score (0-1), proportion_score (0-1), issues list
4. Scores blend into the overall comparison score at 45% weight

**API:**
```python
from app.agents import verify_dxf_with_gemini

result = await verify_dxf_with_gemini(
    product_image=opencv_image,      # numpy array from cv2.imread()
    dxf_raster=opencv_dxf_image,     # numpy array from _rasterize_dxf()
    furniture_type="rectangular_table",
    page_dimensions={"width_cm": 120, "depth_cm": 80, "overall_height_cm": 75},
)
# Returns: {shape_match, component_score, proportion_score, issues, explanation, performed}
```

**Requirements:**
- `GEMINI_API_KEY` environment variable set
- `gemini-2.5-flash` model (or override via `GEMINI_OCR_MODEL`)
- OpenCV (`cv2`) installed
- `httpx` installed

**Key difference from pixel-based comparison:**
Gemini understands both a photo and a line drawing as different representations of the same thing. It doesn't compare pixels — it compares meaning. A round table DXF next to a round table photo scores 1.0 regardless of shadows, texture, or perspective differences.
